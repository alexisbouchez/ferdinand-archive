To get started on our managed platform, create an account in the [dashboard](https://ferdinand.email/auth/sign_up). 

Then, you will be able to configure your domails, API keys, and start sending emails. 

## Frequently Asked Questions

<Accordion title="Is it open-source?">
  Yes, Ferdinand is open-source. 
  
  You can find the source code on [GitHub](
    https://github.com/valyentdev/ferdinand
  ) under the [AGPL V3 License](https://github.com/valyentdev/ferdinand/blob/main/LICENSE.md).
</Accordion>
<Accordion title="Can I contribute?">
  Yes, you can! And we would love your help.

  We are always looking for contributors to help us improve the project. 

  You can find take a look at the [open issues](https://github.com/valyentdev/ferdinand/issues) and submit a pull request.

  Or you can join our [Discord server](https://discord.gg/DuW5uQCtZj) to chat with us about how you can help.
</Accordion>
<Accordion title="How can I report a bug?">
  You can report a bug by opening an issue on [the GitHub repository](
    https://github.com/valyentdev/ferdinand
  ).
</Accordion>
<Accordion title="Is it suited to marketing emails?">
  Ferdinand is designed for transactional emails, not marketing emails. 

  If you are looking for a marketing email service, we recommend [Mailchimp](https://mailchimp.com).
</Accordion>
